# Dungeon MVP — 数值设计文档

## 行动系统

所有行动有耗时（duration），AV = 反应时 + 耗时，作为单一值入队倒计时。
av_remaining 递减至 0 时自动执行。行动队列为全局单队列，玩家与怪物混排。

### 行动定义

| 行动 | 耗时 | 说明 |
|---|---|---|
| 移动 | 300 | 朝相邻地板格走一格 |
| 攻击（近战） | 300 | 朝相邻怪物格子移动时触发，共享移动的耗时 |
| 技能 | 600 | 释放职业技能 |
| 等待 | 800 | 原地消耗一回合 |
| 怪物追击 | 250 | 怪物向玩家移动或近战攻击 |
| 怪物逃跑 | 250 | 怪物朝远离玩家方向移动 |
| 怪物游荡 | 500 | 怪物随机移动 |

### 反应时与耗时修正

从行动确认到实际执行之间的延迟。由敏捷决定：

```
反应时 = max(100 - 敏捷 × 3, 20)
耗时修正系数 = max(1.0 - 敏捷 × 0.02, 0.5)
AV = 反应时 + 耗时 × 修正系数
```

| 敏捷 | 反应时 | 耗时系数 |
|---|---|---|
| 3（哥布林） | 91 | ×0.94 |
| 4（蝎子） | 88 | ×0.92 |
| 5（老鼠） | 85 | ×0.90 |
| **10（玩家）** | **70** | **×0.80** |
| 20 | 40 | ×0.60 |
| 25+ | 20（反应时下限） | ×0.50（耗时下限） |

### 行动队列推进

`advance_action_queue()` 每帧由主循环调用：
1. 查最小 av_remaining（next_event_distance）
2. 所有条目同步推进该距离
3. 弹出 av ≤ 0 的条目，逐个执行
4. 每个条目执行前保活检查（check_condition），条件不满足则丢弃
5. 每执行一个条目后重建碰撞图，防止实体重叠

### 玩家确认后持续推进

玩家确认行动后，`advance_until_player_acted()` 循环调用 advance_action_queue 直到玩家的行动被执行完毕（或队列空）。

### 怪物决策（并行 Schedule）

每帧一次，通过 bevy Schedule 并行执行三个决策 system：

```
chase_decision_system  ─┐
flee_decision_system   ─┤ 并发 → arbitration_system → ActionQueue
wander_decision_system ─┘
```

- 每个 system 检查自己的条件，写入独立意图缓冲区
- `arbitration_system` 按优先级排序，合并入 ActionQueue
- 已有队列条目的实体不再重复入队

## 战斗公式

### 近战伤害

```
基础伤害 = max(攻击力 - 目标防御力, 1)
```

攻击力 = 基础攻击 + 装备加成 + Buff 加成（狂暴）

### 暴击

```
暴击判定 = random() < 暴击率
暴击伤害 = 基础伤害 × (1 + 暴击伤害倍率)
```

| 属性 | 玩家基础值 |
|---|---|
| 暴击率 | 5% |
| 暴击伤害 | 50% |

### 技能伤害 / 效果

| 技能 | 效果公式 | MP消耗 | 熟练度加成（每级） |
|---|---|---|---|
| 治愈 | `15 + 法术精通 × 1 + 熟练度 × 3` HP恢复 | 6 | `+3` |
| 护盾 | 防御 `5 + 熟练度 × 2`，持续3s（3000 AV） | 5 | `+2` |
| 狂暴 | 攻击 `5 + 熟练度 × 2`，持续3s（3000 AV） | 5 | `+2` |

### HP/MP 成长

```
最大HP = 20 + 等级 × 5 + 防御 × 2
最大MP = 5 + 等级 × 3 + 法术精通
```

### 装备加成

装备直接持有 ItemStack（不占背包空间），属性加成立即生效：

```
有效攻击 = 基础攻击 + 装备攻击之和 + Buff加成
有效防御 = 基础防御 + 装备防御之和 + Buff加成
```

## Buff 系统

Buff 使用 AV 制时间轴，在 `advance_action_queue` 中与行动队列同步推进（`remaining_av -= dist`）。

| Buff | 来源 | 效果 | 持续 AV |
|---|---|---|---|
| 护盾 | 战士/牧师技能 | 防御 +5 | 3000（≈3s） |
| 狂暴 | 战士技能 | 攻击 +5 | 3000（≈3s） |

## 技能卷轴

### 掉落与获取

技能卷轴是地图上可拾取的物品，每层随机生成 1-3 张。

| 卷轴 | 对应技能 | 楼层分布 |
|------|---------|---------|
| 治愈卷轴 | Heal | F1+ |
| 护盾卷轴 | Shield | F1+ |
| 狂暴卷轴 | Berserk | F1+ |

### 学习机制

- **首次使用卷轴：** 学会对应技能，熟练度 = 1
- **重复使用同种卷轴：** 熟练度 +1（显示为熟练度提升）
- 熟练度上限暂不设（预留）

### 熟练度效果

```
治愈量   = 15 + 法术精通 × 1 + 熟练度 × 3
护盾值   = 5 + 熟练度 × 2
狂暴攻击 = 5 + 熟练度 × 2
```

| 熟练度 | 治愈 | 护盾 | 狂暴 |
|--------|------|------|------|
| 1（初学） | 15 + 法术精通 | +5 | +5 |
| 2（熟悉） | 15 + 法术精通 + 3 | +7 | +7 |
| 3（精通） | 15 + 法术精通 + 6 | +9 | +9 |
| 4（大师） | 15 + 法术精通 + 9 | +11 | +11 |

### 技能来源变更

- ~~职业锁定~~ → 无职业设计，初始无技能
- 所有技能通过卷轴获取
- `Skills` 组件动态扩展（不再是 `PlayerClass::skills()` 硬编码）


## 掉落物系统

### 设计参考

本系统的物品 ID 化、ItemStack、物品注册表和 LootTable 设计参考了 **Minecraft Java Edition 1.16+** 的物品系统架构：

- **Registry 模式**：物品定义存放在 JSON 中，通过 `usize` ID 索引。运行时加载为 `ItemRegistry`（OnceLock 全局单例）
- **ItemStack**：背包中不存完整的物品定义，只存 `(item_id, count)` 对。堆叠上限由物品定义中的 `max_stack` 控制（装备 1，材料 64）
- **LootTable**：怪物挂载 `LootTable` 组件，死亡时掷骰决定掉落。每次 roll 各条目独立判定

### 怪物视觉

每个怪物使用基于实体 ID 的独特色彩（HSV 空间，S/V ≥ 0.7），同类怪物不同个体颜色不同、整体保持鲜艳。

### 怪物行为

| 行为 | 优先级（数值越高越优先） | 触发条件 |
|---|---|---|
| 逃跑 | 200 | HP < 25% |
| 追击 | 100 | 视野范围内看见玩家 |
| 游荡 | 50 | 无条件（非追击/逃跑时） |
| 等待 | 0 | 无条件（兜底） |

追击逻辑：
- 与玩家邻接（含对角） → 近战攻击（伤害 = 怪物攻击 - 玩家有效防御）
- 不与玩家邻接 → A* 寻路（8 方向，OccupancyMap 避让，不检查对角穿墙）朝玩家方向移动一步

逃跑逻辑：
- 朝远离玩家的方向移动

游荡逻辑：
- 随机选择一个方向移动

### 行为约束

- 已有行动在队列中的怪物不再重复入队（`has_entity` 检查）
- 执行前保活检查：追击目标仍在视野？逃跑条件仍满足？目标格仍可通行？
- 每次行动执行后重建碰撞图，确保后续条目知道最新位置

## 投掷物（Batch 1）

石子是首个投掷类消耗品，引入远程攻击和弹道可视化机制。

### 石子

| 属性 | 值 |
|------|-----|
| ID | 18 |
| 来源 | 哥布林掉落（60%, 1-2） + 地面刷新（每层 1-3 堆） |
| 堆叠 | 20 |
| 使用方式 | 背包 `r` 键 → 进入投掷瞄准模式 |

### 投掷瞄准模式

进入瞄准模式后：

- 光标出现（同 `x` 查看模式），范围显示有效射程
- **弹道轨迹**：从玩家到光标沿线每格绘制符号
  - 蓝色 `*` ← 射程内 + 视线畅通（可投掷）
  - 红色 `*` ← 超出射程或视线阻挡（不可投掷）
- 方向键移动光标，`Enter` 确认投掷，`x`/`Esc` 取消
- 确认后移除背包中 1 颗石子

### 射程

切比雪夫距离 ≤ 5 格（`max(dx, dy) ≤ 5`）。

### 视线检查

弹道经过的格中**任一格阻挡视线**（Wall / Stalactite）→ 目标不可选中，轨迹变红。

### 伤害公式

```
石子伤害 = max(基础伤害 + 随机(0..1) − 目标防御, 1)
基础伤害 = 3 + floor(楼层 ÷ 2)
```

| 楼层 | 基础伤害 | 对防御 0 目标 | 对防御 3 哥布林 |
|------|---------|--------------|----------------|
| 1 | 3 | 3-4 | 1 |
| 5 | 5 | 5-6 | 2-4 |
| 10 | 7 | 7-8 | 4-6 |

石子**受装备暴击率加成**（同 G16 修复），但不继承玩家攻击力。

### 设计意图

- 石子 ≈ 一半的近战伤害，但提供**远程 + 视线检查**的战术优势
- 伤害随楼层增长，深层不完全沦为废品
- 弹道轨迹提供终端 Roguelike 少有的**可视化投射**体验
- 视线检查防止"隔墙打人"，弹道对玩家和怪物规则一致
- 语法约定：蓝色 = 弹道/投射物，紫色 = 法术/技能（预留），红色 = 无效

### 生成管线

```
terrain-forge room_accretion（Brogue 风格有机洞穴）
  → detect_cave_regions（BFS 连通区检测）
  → generate_water（2% 噪声种子 → 深水扩散 → 浅水扩散）
  → carve_expand（墙邻接 walkable → 25% 挖成 Floor）
  → generate_stalactites（每房间 ~7%）

```

## 输入系统

独立输入线程 + 主循环非阻塞接收，16ms 限流轮询，50ms 按键去重。

```
输入线程 → mpsc channel → 主循环 try_recv → process_key → advance_and_settle_parallel → render
```

- 输入线程：独立线程 `crossterm::event::poll(16ms)`，50ms 同键去重
- 主循环：`try_recv()` 非阻塞，无输入时 `sleep(1ms)` 防空转
- tap-tap：第一次按键设预览，第二次同键确认入队
- `x` 键：光标查看模式（方向键移动、x/Esc 退出），显示地形和实体信息
- 模态对话框（退出/下楼/背包）：`AtomicBool` 暂停输入线程，主线程直读 stdin
