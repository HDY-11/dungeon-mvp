# Dungeon MVP — 数值设计文档

## 行动系统

所有行动有耗时（duration），AV = 反应时 + 耗时，作为单一值入队倒计时。
av_remaining 递减至 0 时自动执行。行动队列为全局单队列，玩家与怪物混排。

### 行动定义

| 行动 | 耗时 | 说明 |
|---|---|---|
| 移动 | 300 | 朝相邻地板格走一格 |
| 攻击（近战） | 300 | 朝相邻怪物格子移动时触发，共享移动的耗时 |
| 技能 | 600 | 释放职业技能 |
| 等待 | 800 | 原地消耗一回合 |
| 怪物追击 | 250 | 怪物向玩家移动或近战攻击 |
| 怪物逃跑 | 250 | 怪物朝远离玩家方向移动 |
| 怪物游荡 | 500 | 怪物随机移动 |

### 反应时与耗时修正

从行动确认到实际执行之间的延迟。由敏捷决定：

```
反应时 = max(100 - 敏捷 × 3, 20)
耗时修正系数 = max(1.0 - 敏捷 × 0.02, 0.5)
AV = 反应时 + 耗时 × 修正系数
```

| 敏捷 | 反应时 | 耗时系数 |
|---|---|---|
| 3（哥布林） | 91 | ×0.94 |
| 4（蝎子） | 88 | ×0.92 |
| 5（老鼠） | 85 | ×0.90 |
| **10（玩家）** | **70** | **×0.80** |
| 20 | 40 | ×0.60 |
| 25+ | 20（反应时下限） | ×0.50（耗时下限） |

### 行动队列推进

`advance_action_queue()` 每帧由主循环调用：
1. 查最小 av_remaining（next_event_distance）
2. 所有条目同步推进该距离
3. 弹出 av ≤ 0 的条目，逐个执行
4. 每个条目执行前保活检查（check_condition），条件不满足则丢弃
5. 每执行一个条目后重建碰撞图，防止实体重叠

### 玩家确认后持续推进

玩家确认行动后，`advance_until_player_acted()` 循环调用 advance_action_queue 直到玩家的行动被执行完毕（或队列空）。

### 怪物决策（并行 Schedule）

每帧一次，通过 bevy Schedule 并行执行三个决策 system：

```
chase_decision_system  ─┐
flee_decision_system   ─┤ 并发 → arbitration_system → ActionQueue
wander_decision_system ─┘
```

- 每个 system 检查自己的条件，写入独立意图缓冲区
- `arbitration_system` 按优先级排序，合并入 ActionQueue
- 已有队列条目的实体不再重复入队

## 战斗公式

### 近战伤害

```
基础伤害 = max(攻击力 - 目标防御力, 1)
```

攻击力 = 基础攻击 + 装备加成 + Buff 加成（狂暴）

### 暴击

```
暴击判定 = random() < 暴击率
暴击伤害 = 基础伤害 × (1 + 暴击伤害倍率)
```

| 属性 | 玩家基础值 |
|---|---|
| 暴击率 | 5% |
| 暴击伤害 | 50% |

### 技能伤害 / 效果

| 技能 | 效果公式 | MP消耗 |
|---|---|---|
| 火球 | `15 + 法术精通 × 0.5` 伤害，目标须邻接 | 10 |
| 治愈 | `15 + 法术精通 × 1` HP恢复 | 6 |
| 护盾 | 防御 +5，持续3回合 | 5 |
| 狂暴 | 攻击 +5，持续3回合 | 5 |


### 装备加成

装备直接持有 ItemStack（不占背包空间），属性加成立即生效：

```

有效防御 = 基础防御 + 装备防御之和 + Buff加成
```

## 掉落物系统

### 设计参考

本系统的物品 ID 化、ItemStack、物品注册表和 LootTable 设计参考了 **Minecraft Java Edition 1.16+** 的物品系统架构：

- **Registry 模式**：物品定义存放在 JSON 中，通过 `usize` ID 索引。运行时加载为 `ItemRegistry`（OnceLock 全局单例）
- **ItemStack**：背包中不存完整的物品定义，只存 `(item_id, count)` 对。堆叠上限由物品定义中的 `max_stack` 控制（装备 1，材料 64）
- **LootTable**：怪物挂载 `LootTable` 组件，死亡时掷骰决定掉落。每次 roll 各条目独立判定

### 背包

- 按 `e` 打开背包（双栏界面：左侧背包+装备，右侧地面物品）
- 按 `g` 快速拾取脚下所有物品
- 相同物品自动堆叠至 `max_stack`
- 装备物品不占背包空间，存储在装备槽中
- `←`/`→` 切换左右栏，`↑`/`↓` 移动，`0-9` 或 `a-z` 快捷选中
- `Enter` 查看详情，`e` 装备，`d` 丢弃，`u` 卸载装备

### 怪物行为

| 行为 | 优先级（数值越高越优先） | 触发条件 |
|---|---|---|
| 逃跑 | 200 | HP < 25% |
| 追击 | 100 | 视野范围内看见玩家 |
| 游荡 | 50 | 无条件（非追击/逃跑时） |
| 等待 | 0 | 无条件（兜底） |

追击逻辑：
- 与玩家邻接（含对角） → 近战攻击（伤害 = 怪物攻击 - 玩家有效防御）
- 不与玩家邻接 → A* 寻路（8 方向，对角穿墙角检查）朝玩家方向移动一步

逃跑逻辑：
- 朝远离玩家的方向移动

游荡逻辑：
- 随机选择一个方向移动

### 行为约束

- 已有行动在队列中的怪物不再重复入队（`has_entity` 检查）
- 执行前保活检查：追击目标仍在视野？逃跑条件仍满足？目标格仍可通行？
- 每次行动执行后重建碰撞图，确保后续条目知道最新位置

### 生成管线

```
terrain-forge room_accretion（Brogue 风格有机洞穴）
  → detect_cave_regions（BFS 连通区检测）
  → generate_water（2% 噪声种子 → 深水扩散 → 浅水扩散）
  → carve_expand（墙邻接 walkable → 25% 挖成 Floor）
  → generate_stalactites（每房间 ~7%）
  → ensure_connectivity（2 格宽醉汉游走）
```

房间之间不再用 L 形走廊，由连通性保障机制处理。
第一间为玩家出生点，楼梯选距出生点最远的房间中心。

## 输入系统

独立输入线程 + 主循环非阻塞接收，16ms 限流轮询，50ms 按键去重。

```
输入线程 → mpsc channel → 主循环 try_recv → process_key → advance_and_settle_parallel → render
```

- 输入线程：独立线程 `crossterm::event::poll(16ms)`，50ms 同键去重
- 主循环：`try_recv()` 非阻塞，无输入时 `sleep(1ms)` 防空转
- tap-tap：第一次按键设预览，第二次同键确认入队
- 模态对话框（退出/下楼/背包）：`AtomicBool` 暂停输入线程，主线程直读 stdin
