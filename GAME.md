# Dungeon MVP — 数值设计文档

## Gm1 行动系统

所有行动有耗时（duration），AV = 反应时 + 耗时，作为单一值入队倒计时。
av_remaining 递减至 0 时自动执行。行动队列为全局单队列，玩家与怪物混排。

### 行动定义

| 行动 | 耗时 | 说明 |
|---|---|---|
| 移动 | 300 [⃞直觉: DCSS 移动≈1s, 终端节奏快 3x] | 朝相邻地板格走一格 |
| 攻击（近战） | 300 [⃞计算: 攻击是移动的副效应, 共享耗时] | 朝相邻怪物格子移动时触发，共享移动的耗时 |
| 投掷 | 190 [⃞试调: 原 400→怪物先执行致投掷落空→改 190→验证通过, 见 G22] | 远程投掷消耗品（如石子） |
| 技能 | 600 [⃞直觉: 约为移动 2x, 施法需站桩惩罚] | 释放职业技能 |
| 等待 | 800 [⃞直觉: 最慢行动, 鼓励主动不等待] | 原地消耗一回合 |
| 怪物追击 | 250 [⃞直觉: 快于玩家移动, 制造被追上压力] | 怪物向玩家移动或近战攻击 |
| 怪物逃跑 | 250 [⃞计算: 与追击同速, 保持行动一致性] | 怪物朝远离玩家方向移动 |
| 怪物游荡 | 500 [⃞直觉: 游荡是探索行为, 不应对玩家构成压力] | 怪物随机移动 |

### 反应时与耗时修正

从行动确认到实际执行之间的延迟。由敏捷决定：

```
反应时 = max(100 - 敏捷 × 3, 20)  [⃞直觉: 100 基准/3 系数/20 下限均为估值]
耗时修正系数 = max(1.0 - 敏捷 × 0.02, 0.5)  [⃞直觉: 每点敏捷-2%, 50% 下限防极端, 系数与下限均为直觉]
AV = 反应时 + 耗时 × 修正系数
```

| 敏捷 | 反应时 | 耗时系数 |
|---|---|---|
| 3（哥布林） | 91 | ×0.94 |
| 4（蝎子） | 88 | ×0.92 |
| 5（老鼠） | 85 | ×0.90 |
| **10（玩家）** | **70** | **×0.80** |
| 20 | 40 | ×0.60 |
| 25+ | 20（反应时下限） | ×0.50（耗时下限） |

### 行动队列推进

`advance_action_queue()` 每帧由主循环调用：
1. 查最小 av_remaining（next_event_distance）
2. 所有条目同步推进该距离
3. 弹出 av ≤ 0 的条目，逐个执行
4. 每个条目执行前保活检查（check_condition），条件不满足则丢弃
5. 每执行一个条目后重建碰撞图，防止实体重叠

### 玩家确认后持续推进

玩家确认行动后，`advance_until_player_acted()` 循环调用 advance_action_queue 直到玩家的行动被执行完毕（或队列空）。

### 怪物决策（并行 Schedule）

每帧一次，通过 bevy Schedule 并行执行三个决策 system：

```
chase_decision_system  ─┐
flee_decision_system   ─┤ 并发 → arbitration_system → ActionQueue
wander_decision_system ─┘
```

- 每个 system 检查自己的条件，写入独立意图缓冲区
- `arbitration_system` 按优先级排序，合并入 ActionQueue
- 已有队列条目的实体不再重复入队

## Gm2 战斗公式

### 近战伤害

```
基础伤害 = max(攻击力 - 目标防御力, 1)  [⃞直觉: 经典差值公式, 1 保底防零伤害]
```

攻击力 = 基础攻击 + 装备加成 + Buff 加成（狂暴）

### 暴击

```
暴击判定 = random() < 暴击率
暴击伤害 = 基础伤害 × (1 + 暴击伤害倍率)
```

| 属性 | 玩家基础值 |
|---|---|
| 暴击率 | 5% [⃞直觉: Dota2 PA 暴击 15%, 终端低频战斗折半再折半] |
| 暴击伤害 | 50% [⃞直觉: 传统 150%, 取一半防爆发过强] |

### 技能伤害 / 效果

| 技能 | 效果公式 | MP消耗 | 熟练度加成（每级） |
|---|---|---|---|
| 治愈 | `15 + 法术精通 × 1 + 熟练度 × 3` HP恢复 [⃞直觉: 15 基值+法术精通系数 1+熟练度系数 3, 均未校准] | 6 [⃞直觉: 约 1/3 MP 池, 限制连续施法] | `+3` |
| 护盾 | 防御 `5 + 熟练度 × 2`，持续3s（3000 AV） [⃞直觉: 5 基值, 持续 3000AV 约 3 次行动] | 5 [⃞直觉: 略低于治愈, 防御性技能轻耗] | `+2` |
| 狂暴 | 攻击 `5 + 熟练度 × 2`，持续3s（3000 AV） [⃞直觉: 同护盾结构, 对称设计] | 5 [⃞直觉: 同护盾] | `+2` |

```
升到下一级所需经验 = 25 × 等级^1.5 + 10 × 等级  [⃞直觉: 多项式增长, 1.5 次方≈RPG 常规曲线; 系数 25/10 是估值]
```

|---|---|
| 1→2 | 35 |
| 2→3 | 91 |
| 3→4 | 160 |
| 4→5 | 240 |
| 5→6 | 330 |
| 6→7 | 427 |
| 7→8 | 533 |
| 8→9 | 646 |
| 9→10 | 765 |
| 10→11 | 891 |

### 怪物经验

```
老鼠经验 = round(6 + (楼层-1) × 6 × 0.5)  [⃞直觉: 基础 6, 每层+3, 系数 0.5 防增速过快]
```

|---|---|---|---|

### HP/MP 成长

```
最大HP = 20 + 等级 × 5 + 防御 × 2  [⃞直觉: 见 Gm4 属性表]
最大MP = 5 + 等级 × 3 + 法术精通  [⃞直觉: 见 Gm4 属性表]
```

### 装备槽位

| 槽位 | 可装备 | 说明 |
|------|--------|------|
| 主手 (`MainHand`) | 武器 | 攻击力的主要来源 |
| 副手 (`OffHand`) | 盾牌、投掷物 | 投掷时自动装副手，消耗副手物品 |
| 防具 (`Armor`) | 盔甲 | 提供防御加成 |
| 戒指 (`Ring`) | 戒指 | 提供属性加成 |

### 装备加成

装备直接持有 ItemStack（不占背包空间），属性加成立即生效：

```
有效攻击 = 基础攻击 + 装备攻击之和 + Buff加成
有效防御 = 基础防御 + 装备防御之和 + Buff加成
```

投掷时主手武器不参与攻击力计算，投掷伤害仅由投掷物自身决定。

## Gm6 技能卷轴

### 掉落与获取

技能卷轴是地图上可拾取的物品，每层随机生成 1-3 张。

| 卷轴 | 对应技能 | 楼层分布 |
|------|---------|---------|
| 治愈卷轴 | Heal | F1+ |
| 护盾卷轴 | Shield | F1+ |
| 狂暴卷轴 | Berserk | F1+ |

### 学习机制

- **首次使用卷轴：** 学会对应技能，熟练度 = 1
- **重复使用同种卷轴：** 熟练度 +1（显示为熟练度提升）
- 熟练度上限暂不设（预留）

```
治愈量   = 15 + 法术精通 × 1 + 熟练度 × 3  [⃞直觉: 15 基值, 法术精通系数 1, 熟练度系数 3]

狂暴攻击 = 5 + 熟练度 × 2  [⃞直觉: 同护盾对称]
```

| 熟练度 | 治愈 | 护盾 | 狂暴 |
|--------|------|------|------|
| 1（初学） | 15 + 法术精通 | +5 | +5 |
| 2（熟悉） | 15 + 法术精通 + 3 | +7 | +7 |
| 3（精通） | 15 + 法术精通 + 6 | +9 | +9 |
| 4（大师） | 15 + 法术精通 + 9 | +11 | +11 |

### 技能来源变更

- ~~职业锁定~~ → 无职业设计，初始无技能
- 所有技能通过卷轴获取
- `Skills` 组件动态扩展（不再是 `PlayerClass::skills()` 硬编码）


## Gm7 掉落物系统

### 设计参考

本系统的物品 ID 化、ItemStack、物品注册表和 LootTable 设计参考了 **Minecraft Java Edition 1.16+** 的物品系统架构：

- **Registry 模式**：物品定义存放在 JSON 中，通过 `usize` ID 索引。运行时加载为 `ItemRegistry`（OnceLock 全局单例）
- **ItemStack**：背包中不存完整的物品定义，只存 `(item_id, count)` 对。堆叠上限由物品定义中的 `max_stack` 控制（装备 1，材料 64）
- **LootTable**：怪物挂载 `LootTable` 组件，死亡时掷骰决定掉落。每次 roll 各条目独立判定

### 背包

- 按 `e` 打开背包（双栏界面：左侧背包+装备，右侧地面物品）

### 怪物视觉

每个怪物使用基于实体 ID 的独特色彩（HSV 空间，S/V ≥ 0.7），同类怪物不同个体颜色不同、整体保持鲜艳。

### 怪物行为

| 行为 | 优先级（数值越高越优先） | 触发条件 |
|---|---|---|
| 逃跑 | 200 | HP < 25% |
| 追击 | 100 | 视野范围内看见玩家 |
| 游荡 | 50 | 无条件（非追击/逃跑时） |
| 等待 | 0 | 无条件（兜底） |

追击逻辑：
- 与玩家邻接（含对角） → 近战攻击（伤害 = 怪物攻击 - 玩家有效防御）
- 不与玩家邻接 → A* 寻路（8 方向，OccupancyMap 避让，不检查对角穿墙）朝玩家方向移动一步

逃跑逻辑：
- 朝远离玩家的方向移动

游荡逻辑：
- 随机选择一个方向移动

### 行为约束

- 已有行动在队列中的怪物不再重复入队（`has_entity` 检查）
- 执行前保活检查：追击目标仍在视野？逃跑条件仍满足？目标格仍可通行？
- 每次行动执行后重建碰撞图，确保后续条目知道最新位置

## Gm9 投掷物（Batch 1）

石子是首个投掷类消耗品，引入远程攻击和弹道可视化机制。

### 石子

| 属性 | 值 |
|------|-----|
| ID | 18 |
| 来源 | 哥布林掉落（60%, 1-2） + 地面刷新（每层 1-3 堆） |
| 堆叠 | 20 |
| 使用方式 | `t` 键（投掷选单）或背包 `r` 键 → 进入投掷瞄准模式 |

### 投掷瞄准模式

进入瞄准模式后：

- 光标出现（同 `x` 查看模式），范围显示有效射程
- **弹道轨迹**：从玩家到光标沿线每格绘制符号
  - 蓝色 `*` ← 射程内 + 视线畅通（可投掷）
  - 红色 `*` ← 超出射程或视线阻挡（不可投掷）
- 方向键移动光标，`Enter` 确认投掷，`x`/`Esc` 取消
- 确认后投掷行动入队（耗时 190ms AV），等待倒计时结束后执行
- 执行时消耗副手槽 1 颗石子

### 射程

切比雪夫距离 ≤ 5 格（`max(dx, dy) ≤ 5`）。

### 视线检查

弹道经过的格中**任一格阻挡视线**（Wall / Stalactite）→ 目标不可选中，轨迹变红。

### 伤害公式

```
石子伤害 = max(基础伤害 + 随机(0..1) − 目标防御, 1)  [⃞计算: 见 Gm2 通用战斗公式框架, 投掷继承攻击流程]
基础伤害 = 3 + floor(楼层 ÷ 2)  [⃞直觉: 3 基值≈近战一半, 每 2 层+1 防深层贬值]
```

| 楼层 | 基础伤害 | 对防御 0 目标 | 对防御 3 哥布林 |
|------|---------|--------------|----------------|
| 1 | 3 | 3-4 | 1 |
| 5 | 5 | 5-6 | 2-4 |
| 10 | 7 | 7-8 | 4-6 |

石子**受装备暴击率加成**（同 G16 修复），但不继承玩家攻击力。

### 设计意图

- 石子 ≈ 一半的近战伤害，但提供**远程 + 视线检查**的战术优势
- 伤害随楼层增长，深层不完全沦为废品
- 弹道轨迹提供终端 Roguelike 少有的**可视化投射**体验
- 视线检查防止"隔墙打人"，弹道对玩家和怪物规则一致
- 语法约定：蓝色 = 弹道/投射物，紫色 = 法术/技能（预留），红色 = 无效

### 生成管线

```
terrain-forge room_accretion（Brogue 风格有机洞穴）
  → detect_cave_regions（BFS 连通区检测）
  → generate_water（2% 噪声种子 → 深水扩散 → 浅水扩散）
  → carve_expand（墙邻接 walkable → 25% 挖成 Floor）
  → generate_stalactites（每房间 ~7%）
  → ensure_connectivity（2 格宽醉汉游走）
  → ensure_spawn_accessible（出生点被困时醉汉游走打破）
```

房间之间不再用 L 形走廊，由连通性保障机制处理。

## Gm11 输入系统

独立输入线程 + 主循环非阻塞接收，16ms 限流轮询，50ms 按键去重。

```
输入线程 → mpsc channel → 主循环 try_recv → process_key → advance_and_settle_parallel → render
```

- 输入线程：独立线程 `crossterm::event::poll(16ms)`，50ms 同键去重
- 主循环：`try_recv()` 非阻塞，无输入时 `sleep(1ms)` 防空转
- tap-tap：第一次按键设预览，第二次同键确认入队
- `x` 键：光标查看模式（方向键移动、x/Esc 退出），显示地形和实体信息
- 模态对话框（退出/下楼/背包）：`AtomicBool` 暂停输入线程，主线程直读 stdin
